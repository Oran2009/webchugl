What is WebChuGL?

github | docs

WebChuGL brings ChuGL, the real-time graphics framework for the ChucK programming language, to the web browser! ChuGL’s C++ source code has been compiled with Emscripten to WebAssembly (WASM) to render via the WebGPU API and run audio through an AudioWorkletNode. With near-native performance, WebChuGL runs on modern desktop browsers as well as tablets and mobile devices!

WebChuGL builds on WebChucK by adding ChuGL’s scenegraph, materials, textures, shaders, post-processing, and the full GGen (graphics generator) ecosystem — all tightly synchronized to ChucK’s strongly-timed audio engine. Bring together real-time graphics, sound synthesis, and the interconnectivity of the web to create new audiovisual experiences and creative workflows!

Features

🎨

WebGPU Rendering

Hardware-accelerated graphics via the modern WebGPU API! Scenegraph, PBR materials, textures, custom shaders, post-processing effects, and more.

🎵

Strongly-Timed Audio

ChucK’s powerful audio engine runs via AudioWorklet with lock-free ring buffers for sample-accurate timing, synthesis, effects, and music.

🔌

ChuGin Ecosystem

Load ChuGins (ChucK plugins) compiled as WebAssembly side modules — extend the language with custom UGens and libraries!

🎮

Input & Sensors

Keyboard, mouse, gamepad, MIDI, accelerometer, gyroscope, drag-and-drop files, speech recognition, MediaPipe hand tracking, and more!

🌐

JS ↔ ChucK Bridge

Full bidirectional communication between JavaScript and ChucK! Set and get globals, listen for events, load audio dynamically, and more.

⚙️

Configurable Audio

Set sample rate, channel count, and more via URL params (?srate=44100&out=4) or the init config.

Getting Started

All you need is a browser with WebGPU support (Chrome 113+, Edge 113+, Firefox Nightly). Create an index.html:

<canvas id="canvas"></canvas>
<script type="module">
    import ChuGL from 'https://cdn.jsdelivr.net/npm/webchugl/+esm';

    var ck = await ChuGL.init({
        canvas: document.getElementById('canvas'),
    });

    ck.runCode('SinOsc s => dac; while(true) GG.nextFrame() => now;');
</script>

See the full getting started guide for self-hosted builds, deployment, cross-origin isolation, and building from source.
